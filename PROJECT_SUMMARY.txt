PROBLEM ADDRESSED:
------------------
BTC and other symbols have original signals with TP1/TP2/Entry/SL, but the
trading mode is "trail-only" (no fixed TP, only trailing SL). Statistics were
showing SL hits instead of actual TP hits from the original signal.

SOLUTION IMPLEMENTED:
---------------------
Client-side SignalTracker module that independently monitors price action and
tracks original signal outcomes (TP1/TP2/SL hits) based on candle data,
completely separate from trade execution results.

HOW IT WORKS
================================================================================

1. SIGNAL RECEIVED
   Dashboard receives BUY/SELL signal from bridge API

2. SIGNAL RECORDED
   Signal stored in SignalTracker with entry, SL, TP1, TP2

3. PRICE MONITORED
   New candle data checked every 60 seconds

4. OUTCOME DETECTED
   System detects when price hits TP1, TP2, or SL

5. STATS UPDATED
   Original signal statistics reflect TRUE performance

EXAMPLE (BTCUSD Trail-Only Mode):
----------------------------------
Original Signal: BUY @ 65000
├─ TP1: 65150 (+150 pips)
├─ TP2: 65300 (+300 pips)
└─ SL:  64850 (-150 pips)

Trade Execution (trail-only):
└─ Exits at 65080 (+80 pips)

Original Signal Tracking:
└─ Monitors price independently
└─ If price hits 65150 → Records TP1 hit ✓
└─ If price hits 65300 → Records TP2 hit ✓
└─ If price hits 64850 → Records SL hit

Dashboard Statistics:
└─ Shows: TP1 hit (TRUE signal performance)
└─ NOT: SL hit (which was execution result)

DEBUGGING
================================================================================

CONSOLE LOG PATTERNS:
✓ [SignalTracker] [ADD] - New signal recorded
✓ [SignalTracker] [HIT_TP1/TP2/SL] - Outcome detected
✓ [SignalTracker] [OUTCOME] - Signal outcome finalized
⚠ [SignalTracker] [EXPIRED] - Signal expired (500+ minutes)
✗ [SignalTracker ERROR] - Error occurred

COMMON ISSUES:
- Statistics showing incorrect values → Check console for errors
- Signals not tracking outcomes → Verify TP/SL values are non-zero
- Console errors appearing → Check network connectivity
- History table not updating → Wait for new BUY/SELL signals

QUICK FIXES:
# Clear all tracked signals
localStorage.removeItem('ddd_tracked_signals');
location.reload();

# View all tracked signals
JSON.parse(localStorage.getItem('ddd_tracked_signals'));
